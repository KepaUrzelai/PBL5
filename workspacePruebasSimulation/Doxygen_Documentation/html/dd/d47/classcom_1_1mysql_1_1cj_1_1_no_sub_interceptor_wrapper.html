Detailed Description

Wraps query interceptors during driver startup so that they don't produce different result sets than we expect.

◆ executeTopLevelOnly()

boolean com.mysql.cj.NoSubInterceptorWrapper.executeTopLevelOnly ( )

Should the driver execute this interceptor only for the "original" top-level query, and not put it in the execution path for queries that may be executed from other interceptors?

If an interceptor issues queries using the connection it was created for, and does not return true for this method, it must ensure that it does not cause infinite recursion.

Returns
true if the driver should ensure that this interceptor is only executed for the top-level "original" query.
